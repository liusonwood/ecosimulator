# EcoSim - 框架设计与开发说明 (v3.2)

## 1. 运行环境与约束
- **零服务器依赖**：项目支持通过 `file://` 协议直接打开运行，适合离线演示。
- **非模块化开发**：为了保证兼容性，不使用 `import/export` 语法。所有库（Vue 3, GPU.js, Chart.js, Lucide）均通过 `lib/` 文件夹下的本地脚本引入。
- **单入口逻辑**：核心逻辑集中在 `main.js` 中，通过 `CONFIG` 对象统一管理所有生物学参数和版本信息。

## 2. 文件组成
1.  **`index.html`**：主入口。定义了 16:9 的 Glassmorphism UI 结构及 Vue 模板。
2.  **`style.css`**（新增）：从 HTML 拆分出的样式表，负责视觉特效、布局和自定义滚动条。
3.  **`main.js`**：核心引擎。包含：
    *   `EcoSimulator` 类：计算中心。
    *   GPU/Worker/Serial 渲染内核。
    *   Vue 3 响应式状态管理与生命周期。
4.  **`lib/`**：存放所有第三方依赖库。

## 3. 核心功能实现
- **高性能计算**：优先调用 WebGL (GPU.js) 进行并行演替计算。若环境不支持，则自动降级为多线程 Web Workers 或串行 CPU。
- **生物学模型**：基于 Lotka-Volterra 竞争模型，结合高斯扩散核和演替阈值限制。
- **版本控制**：在 `CONFIG.version` 中定义版本号，实现控制台日志与 UI 显示的自动化同步。

## 4. 扩展与维护
- **修改参数**：直接在 `main.js` 顶部的 `CONFIG` 对象中修改增长率、竞争系数等。
- **增加物种**：需同步更新 `CONFIG` 中的生物参数数组、图标及颜色映射。
