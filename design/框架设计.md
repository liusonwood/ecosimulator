# 任务：开发可本地双击运行的 Vue 3 + GPU.js 生态模拟器

- **所有逻辑必须封装在单一脚本或通过全局变量 Vue、GPU 交互，绝对不能出现 export default 或 import 语句 ！！！！！**

## 1. 运行环境约束 (非常重要)
- **零服务器依赖**：项目必须支持通过 `file://` 协议直接打开（双击 index.html 运行）。
- **非模块化导入**：请勿使用 `type="module"` 或 `import/export` 语法。
- **库引入**：请在 HTML 中通过 `<script src="./lib/xxx.js">` 方式引入 Vue 3, GPU.js 和 Chart.js 的本地副本。

## 2. 核心功能要求
- **Vue 3 逻辑**：使用 `Vue.createApp({...})` 语法。负责解析我提供的 CSV 参数，并管理 UI 状态（启动、暂停、重置）。
- **GPU 加速引擎**：
    - 使用 `new GPU()` 创建实例。
    - 编写 Kernel 处理物种的【扩散 (Convolution)】和【Lotka-Volterra 竞争】。
    - 确保 Kernel 输入支持从 Vue 的响应式数据中提取的 TypedArrays。
- **环境地图**：初始化一个二维数组代表土壤厚度，并映射到 GPU 计算中。
- **手动交互**：在 Canvas 上实现点击拾取坐标，调用接口修改 GPU 内部的矩阵数值以模拟“火灾”或“干扰”。

## 3. 文件结构建议
请提供以下文件的完整代码：
1. `index.html`: 包含 UI 结构和 Canvas 容器。
2. `main.js`: 包含所有生态逻辑和 GPU Kernel 定义。

## 4. 特别逻辑实现
- **土壤乘子**：实现你之前看到的“物种特异性土壤厚度乘子”逻辑。
- **可视化**：每一帧计算后，将结果绘制到 Canvas。使用 Chart.js 绘制种群数量动态曲线。
